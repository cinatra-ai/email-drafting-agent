## Directory Purposes

**`cinatra/`:**
- Purpose: Cinatra platform sidecars — the machine-readable extension contract.
- Contains: `oas.json` (agent flow spec). Workflow extensions would contain `workflow.bpmn` here instead.
- Key files: `cinatra/oas.json`

**`skills/email-drafting/`:**
- Purpose: Mountable LLM skill providing step-by-step prompt instructions.
- Contains: `SKILL.md` with YAML front matter and Markdown instructions consumed by the LLM at inference time.
- Key files: `skills/email-drafting/SKILL.md`

**`.github/workflows/`:**
- Purpose: GitHub Actions CI/CD. `ci.yml` runs on every push/PR to `main`; `release.yml` handles publishing.
- Contains: Two workflow files. The `kind-gates` job in `ci.yml` calls `extension-kind-gate.mjs`.
- Key files: `.github/workflows/ci.yml`, `.github/workflows/release.yml`

**`src/` (declared, not present):**
- Purpose: TypeScript source files if this agent grows imperative logic.
- Contains: Nothing currently — this is a content-only extension.
- Key files: None tracked.

## Key File Locations

**Entry Points:**
- `cinatra/oas.json`: Runtime flow entry — defines `start_node`, all nodes, and all edges.
- `extension-kind-gate.mjs`: CI entry — `main()` dispatches gate based on `cinatra.kind` in `package.json`.

**Configuration:**
- `package.json`: Package identity (`@cinatra-ai/email-drafting-agent`), version, license, and `cinatra` block (kind, apiVersion, agentDependencies).
- `tsconfig.json`: TypeScript compiler options targeting `src/` with `outDir: "dist"`.
- `.npmrc`: npm registry and auth configuration (do not read — may contain tokens).

**Core Logic:**
- `cinatra/oas.json`: All flow logic (nodes, edges, LLM prompt, HITL gate, data wiring).
- `skills/email-drafting/SKILL.md`: Extended agent instructions and MCP tool usage.

**CI Gate:**
- `extension-kind-gate.mjs`: Self-contained validator (~390 lines). Exports `validateAgent`, `validateWorkflow`, `validateBpmnSanity`, `findWorkflowSidecars`, `runGate`, `parseArgs`.

## Naming Conventions

**Files:**
- Cinatra sidecars: `cinatra/oas.json` (agent) or `cinatra/workflow.bpmn` (workflow) — exactly one sidecar per kind.
- Skills: `skills/<skill-name>/SKILL.md` — kebab-case directory, uppercase `SKILL.md`.
- CI gate: `extension-kind-gate.mjs` — kebab-case, `.mjs` ES module extension.
- GitHub workflows: `.github/workflows/<purpose>.yml` — kebab-case.

**Directories:**
- Skill directories: kebab-case matching the skill name (e.g., `email-drafting`).
- Platform sidecar directory: always `cinatra/` (flat, no nesting).

**Package names:**
- Agent packages: `@cinatra-ai/<slug>-agent` (e.g., `@cinatra-ai/email-drafting-agent`).
- Workflow packages: `@cinatra-ai/<slug>-workflow` (enforced by `extension-kind-gate.mjs` regex).

**OAS node IDs:**
- Lowercase kebab-case: `start`, `draft`, `approval_gate`, `end`.

## Where to Add New Code

**New LLM prompt logic:**
- Modify the `system` / `user` fields in `cinatra/oas.json` → `$referenced_components.draft`.
- Do NOT reference any token from `extension-kind-gate.mjs`'s `BANNED_PRIMITIVES` list.

**New skill instructions:**
- Edit `skills/email-drafting/SKILL.md` or add a new skill at `skills/<new-skill-name>/SKILL.md`.

**New flow nodes or edges:**
- Add to `cinatra/oas.json` → `nodes`, `control_flow_connections`, `data_flow_connections`, and `$referenced_components`.

**New TypeScript source (if needed):**
- Create `src/` directory; place `.ts` files there. CI will auto-detect and run `tsc --noEmit`.

**New agent dependency:**
- Add to `package.json` → `cinatra.dependencies` array AND `cinatra.agentDependencies` map.
